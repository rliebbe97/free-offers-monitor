# Actions Needed to Deploy Free Offers Monitor

This is the human-in-the-loop checklist: accounts to create, API keys to obtain, infrastructure to provision, and services to wire up before the system can run end to end. Items are ordered so later steps can consume values generated by earlier ones.

Use `.env.example` in the repo root as the canonical list of environment variables. Every variable below maps to a line in that file.

---

## 1. Anthropic — Claude API key

Required for Tier 1 (Haiku) and Tier 2 (Sonnet) model calls.

1. Sign up at https://console.anthropic.com.
2. Add a payment method and set a **monthly spend limit** (recommended: start at $25 until you have real volume data from `ai_calls`).
3. Create an API key: Settings → API Keys → Create Key. Name it `free-offers-monitor-prod`.
4. Save as `ANTHROPIC_API_KEY`.

Model IDs used (pinned, do not change without running `pnpm eval`):
- `claude-haiku-4-5-20251001`
- `claude-sonnet-4-6`

## 2. Voyage AI — Embedding API key

Required for semantic dedup. Voyage `voyage-3` is 1024-dim, which must match the pgvector column width.

1. Sign up at https://www.voyageai.com.
2. Create an API key in the dashboard.
3. Save as `VOYAGE_API_KEY`.

## 3. Reddit — Descriptive User-Agent (no OAuth)

Reddit ingestion uses **public JSON endpoints** (`/r/<sub>/new.json`, `/r/<sub>/comments/<id>.json`) — no OAuth app, no client ID, no refresh token. This decision is documented in the whitepaper; in short, Reddit's `/prefs/apps` registration is currently effectively closed for new applicants and the redditdev community recommends the public-endpoint path.

1. Identify the Reddit account whose username should appear in the bot's User-Agent string. This is your primary identification to Reddit moderators and any rate-limit appeals.
2. Set `REDDIT_USER_AGENT` in your worker environment, following Reddit's recommended format:

   ```
   REDDIT_USER_AGENT=free-offers-monitor/1.0 (by /u/<your-bot-account>)
   ```

   The default in `config.ts` already uses `/u/Alternative-Owl-7042`. Override only if you fork the project or run under a different account.
3. (Optional) Set `REDDIT_BASE_URL` if you want to point the adapter at a mirror or test fixture host. Defaults to `https://old.reddit.com`.

Rate limits: unauthenticated public JSON endpoints are limited to roughly 10 requests/minute. Steady-state polling is ~2.4 req/min (12 sources × every 5 minutes), comfortably under the cap. The adapter retries with exponential backoff on transient errors and aborts immediately on 404/410.

There is **nothing to register and no credentials to generate** for this step. Compliance with Reddit's Responsible Builder Policy comes from the User-Agent, the polling cadence, the read-only behavior, and the deletion-propagation policy (still TODO — see whitepaper).

## 4. Supabase — Project, schema, and extensions

The database carries all durable state. Provision this before deploying the worker or dashboard.

### 4.1 Create the project

1. Sign up at https://supabase.com.
2. Create a new project. Region: pick one close to Railway's region (e.g. `us-east-1`).
3. Choose a strong database password — you won't need it directly, but save it in your password manager.
4. Wait for provisioning (~2 min).

### 4.2 Collect keys

From Project Settings → API:

- `SUPABASE_URL` = the project URL (e.g. `https://xxxxx.supabase.co`).
- `SUPABASE_SERVICE_ROLE_KEY` = the `service_role` secret key. **Never expose this to the browser.** The worker and Next.js server-only code use it.
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` = the `anon` public key. Used by the dashboard client code.

### 4.3 Enable extensions

In the SQL editor, run:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS pgmq;
CREATE EXTENSION IF NOT EXISTS pg_cron;
```

`pgmq` and `pg_cron` require enabling under Database → Extensions in the Supabase UI first — toggle them on, then re-run the `CREATE EXTENSION` statements if they aren't already active.

### 4.4 Apply the schema

From the repo:

```bash
# paste contents of packages/db/src/schema.sql into the Supabase SQL editor and run
# then apply the seed migration:
# packages/db/src/migrations/001_seed_thebump_sources.sql
```

Or via the Supabase CLI if you prefer a versioned workflow (`supabase db push`).

### 4.5 Seed at least one source

The worker only polls rows present in `sources`. Insert your first subreddit(s):

```sql
INSERT INTO sources (type, identifier, config)
VALUES ('reddit', 'beerbellybabies', '{}'::jsonb),
       ('reddit', 'freebies',        '{}'::jsonb);
```

The TheBump seed migration (`001_seed_thebump_sources.sql`) adds forum sources; apply it if you want TheBump scraping on from day one.

### 4.6 Configure Auth (for dashboard)

1. Authentication → Providers → Email: enable, disable signups (you'll allowlist manually).
2. Authentication → Users → Invite user — add each allowed email.
3. (Optional but recommended) Add an RLS policy on dashboard tables restricting `SELECT` to `auth.email() IN (...)`. The dashboard code assumes this exists.

### 4.7 Schedule validation cron

In the SQL editor:

```sql
SELECT cron.schedule(
  'validate-offers-daily',
  '0 0 * * *',
  $$SELECT pgmq.send('validation_queue', '{"trigger": "cron"}'::jsonb)$$
);
```

(The worker also runs a 10-minute validation loop, so pg_cron is a redundant safety net.)

## 5. Axiom — Structured logging (optional but recommended)

If `AXIOM_TOKEN` is absent the worker falls back to stdout logging, which Railway captures fine. Axiom adds queryable log search.

1. Sign up at https://axiom.co.
2. Create a dataset named `free-offers-monitor` (or whatever you'll set `AXIOM_DATASET` to).
3. Settings → API Tokens → create an ingest token.
4. Save as `AXIOM_TOKEN`, and set `AXIOM_DATASET=free-offers-monitor`.

---

## 6. Railway — Worker deployment

The worker is a long-running Node.js process. Railway is the target; any Node-capable host (Fly, Render, Hetzner + systemd) will also work.

1. Sign up at https://railway.app.
2. New Project → Deploy from GitHub → select this repo.
3. Service settings:
   - Root directory: `apps/worker` (or leave default and set build commands below).
   - Build command: `pnpm install --frozen-lockfile && pnpm build --filter worker`
   - Start command: `pnpm --filter worker start` (or `node apps/worker/dist/index.js`)
   - Port: `3001` (matches `PORT` env default; used for the health endpoint).
4. Environment variables — set all of:
   - `ANTHROPIC_API_KEY`
   - `VOYAGE_API_KEY`
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `REDDIT_USER_AGENT` (optional — falls back to `free-offers-monitor/1.0 (by /u/Alternative-Owl-7042)` if unset; override only if you run under a different bot account)
   - `AXIOM_TOKEN` (optional)
   - `AXIOM_DATASET` (optional, default `free-offers-monitor`)
   - `PORT=3001`
   - `RAILWAY_GIT_COMMIT_SHA` is injected automatically by Railway.
5. Deploy. Tail logs until you see the ingestion / Tier 1 / Tier 2 / validation loops all report "started".
6. Confirm the health endpoint responds: `curl https://<railway-domain>/health` → `200 OK`.

## 7. Vercel — Dashboard deployment

1. Sign up at https://vercel.com and connect your GitHub account.
2. Import the repo → select `apps/dashboard` as the root directory.
3. Framework preset: **Next.js** (auto-detected).
4. Build & Output:
   - Install command: `pnpm install --frozen-lockfile`
   - Build command: `pnpm build --filter dashboard`
   - Output directory: `apps/dashboard/.next` (auto-detected)
5. Environment variables (Settings → Environment Variables, set for Production + Preview):
   - `NEXT_PUBLIC_SUPABASE_URL` = your `SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = the anon key from step 4.2
   - `SUPABASE_SERVICE_ROLE_KEY` = service role key (server-side only; Vercel keeps non-`NEXT_PUBLIC_` vars server-only)
6. Deploy. Visit the assigned URL, log in with an allowlisted email, and confirm the three dashboard routes render:
   - `/dashboard/offers`
   - `/dashboard/review`
   - `/dashboard/ai-logs`
7. (Optional) Attach a custom domain under Settings → Domains.

## 8. GitHub — Repo secrets (only if you add CI)

No CI pipelines ship in the repo today. If you add GitHub Actions for tests or typechecks on PRs, replicate the same secrets into Repository Settings → Secrets → Actions. Do **not** put the service role key in CI unless a job actually needs it.

## 9. Local development

1. `cp .env.example .env.local` at the repo root.
2. Fill in the same values you set in Railway / Vercel (service role key is fine locally; it's in `.gitignore`).
3. `pnpm install`
4. `pnpm dev --filter dashboard` — http://localhost:3000
5. `pnpm dev --filter worker` — starts all four loops in watch mode.
6. `pnpm eval` — runs the Tier 1 classifier + cross-source dedup harness against `evals/labeled-posts.json`. Requires `ANTHROPIC_API_KEY` and `VOYAGE_API_KEY`.

---

## 10. Post-deploy verification checklist

Run through this the first time you deploy:

- [ ] Worker logs show `ingestion loop started`, `tier1 consumer started`, `tier2 consumer started`, `validation loop started`.
- [ ] After ~5 min, `SELECT count(*) FROM posts;` returns > 0.
- [ ] `SELECT count(*) FROM posts WHERE tier0_passed = true;` is non-zero (if zero for multiple cycles, your sources are low-signal — add more subreddits).
- [ ] `SELECT count(*) FROM ai_calls WHERE tier = 1;` grows over time and every row has non-null `cost_usd`.
- [ ] `SELECT count(*) FROM offers;` eventually goes above zero.
- [ ] Dashboard `/dashboard/offers` renders the list.
- [ ] Dashboard `/dashboard/ai-logs` shows Tier 1 and Tier 2 calls with reasonable latencies (Haiku < 2s, Sonnet < 6s typical).
- [ ] After 7 days, `SELECT count(*) FROM verification_log;` is non-zero — validation is running.

## 11. Quick reference — environment variables

| Variable | Where set | Purpose |
|----------|-----------|---------|
| `ANTHROPIC_API_KEY` | Worker (Railway) | Claude API |
| `VOYAGE_API_KEY` | Worker (Railway) | Embedding API |
| `REDDIT_USER_AGENT` | Worker (Railway, optional) | Identifying string sent to Reddit; defaults to `free-offers-monitor/1.0 (by /u/Alternative-Owl-7042)` |
| `REDDIT_BASE_URL` | Worker (optional) | Reddit JSON host; defaults to `https://old.reddit.com` |
| `SUPABASE_URL` | Worker + Dashboard | Postgres connection |
| `SUPABASE_SERVICE_ROLE_KEY` | Worker + Dashboard (server) | Privileged DB access |
| `NEXT_PUBLIC_SUPABASE_URL` | Dashboard (Vercel) | Client Supabase URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Dashboard (Vercel) | Client-side auth |
| `AXIOM_TOKEN` | Worker (Railway, optional) | Log ingestion |
| `AXIOM_DATASET` | Worker (Railway, optional) | Axiom dataset name |
| `PORT` | Worker (Railway) | Health endpoint port (default 3001) |
| `THEBUMP_BASE_URL` | Worker (optional) | Override for TheBump scraper |

Keep `.env` and `.env.local` out of git — `.gitignore` already covers them, but double-check before any `git add -A`.
