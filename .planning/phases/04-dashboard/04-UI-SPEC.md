## Typography

Two weights only: 400 (regular) for all default/body states, 600 (semibold) for all emphasized elements.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 14px | 400 (regular) | 1.5 | Geist Sans |
| Label | 12px | 400 (regular) | 1.3 | Geist Sans |
| Heading | 20px | 600 (semibold) | 1.2 | Geist Sans |
| Display | 28px | 600 (semibold) | 1.1 | Geist Sans |

Notes:
- Monospace values (prompt_version hash, cost_usd, latency_ms cells) use Geist Mono at 12px weight 400.
- Table column headers use Label (12px, weight 400, uppercase, letter-spacing 0.05em).
- Sidebar nav labels use Body (14px, weight 400 default, weight 600 when active).

## Color

All values reference the existing CSS custom properties in `app/globals.css` (OKLCH, neutral palette with dark mode support).

| Role | CSS Variable | Light Mode (OKLCH) | Usage |
|------|-------------|---------------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) — white | Page background, table background |
| Secondary (30%) | `--sidebar` / `--card` | oklch(0.985 0 0) / oklch(1 0 0) — near-white | Sidebar surface, card backgrounds, muted rows |
| Accent (10%) | `--primary` | oklch(0.205 0 0) — near-black | See reserved list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) — red | Reject button, expired status badge only |

Dark mode tokens are already defined in globals.css. Both modes use the same CSS variable names.

Accent (`--primary`) reserved exclusively for:
- Active sidebar navigation indicator (left border + background fill)
- Approve action button (primary variant)
- Pagination current-page indicator
- Login page submit button

Status badge colors (semantic, not accent):
- `active` → green: `bg-green-100 text-green-800` (light) / `bg-green-900/30 text-green-400` (dark)
- `expired` → `bg-muted text-muted-foreground`
- `check_failed` → amber: `bg-amber-100 text-amber-800` (light) / `bg-amber-900/30 text-amber-400` (dark)
- `pending` (review queue) → blue: `bg-blue-100 text-blue-800` (light) / `bg-blue-900/30 text-blue-400` (dark)

## Interaction Contracts

### Navigation
- Fixed sidebar, 240px wide on desktop. Non-collapsible for v1 (mobile: single-column layout, sidebar hidden below `md` breakpoint via `hidden md:flex`).
- Active route highlighted with `--primary` left border (2px) + `--accent` background fill.
- Navigating between sections does not reset URL search params of other sections.

### Offer List
- Page loads server-rendered with default params: `?status=active&page=1`.
- Changing status filter replaces `?status=` param and resets `?page=1`. No client navigation, uses Next.js `<Link>` or `router.push()` with full page transition.
- Pagination: show page numbers 1…N. Show "Previous" / "Next" flanking. Current page uses primary variant. Ellipsis for large page counts.
- Destination URL column: truncated to 40 chars with `…`, full URL in `title` tooltip. External link icon (Lucide `ExternalLink`, 12px) opens in new tab.
- Confidence column: numeric to 2 decimal places (e.g., `0.82`). Values below 0.70 displayed in amber (`text-amber-600`).

### Review Queue
- Table rows are collapsible. Click anywhere on row (outside action buttons) to expand/collapse detail panel.
- Expanded panel shows: extracted offer fields, confidence, AI reasoning string, link to source post.
- Approve/Reject buttons: `variant="outline"` for Reject, `variant="default"` (primary) for Approve. Button disabled + spinner shown during server action in-flight.
- After action: row removed from table with fade-out animation (`tw-animate-css`). Toast shown for 3 seconds.

### AI Call Log
- Default sort: timestamp descending (`?sort=timestamp&dir=desc`).
- Column sort: clicking header toggles asc/desc, updates URL params.
- `prompt_version` displayed as first 7 characters of git hash in monospace font.
- `cost_usd` formatted as `$0.000123` (6 decimal places). `latency_ms` as integer with `ms` suffix.

### Loading States
- All three data views use `<Skeleton>` components matching table structure (row height, column widths) while server components fetch data.
- Login form submit button shows Lucide `Loader2` spinner during authentication request.

### Error States
- Data fetch errors: inline error card above the table with error message copy and "Refresh" link.
- Authentication error (non-allowlisted user): inline error below login form, no toast.
- Server action errors (approve/reject): error toast only, no page reload.

## Focal Point

Primary screen: `/dashboard/offers`

The primary visual anchor is the **offer status badge column** within the data table. The `active` green badge is the highest-contrast element on the dominant white surface and draws the eye first, immediately communicating pipeline health. The page heading "Offers" and the filter bar are secondary anchors, establishing context before the user scans the table.
