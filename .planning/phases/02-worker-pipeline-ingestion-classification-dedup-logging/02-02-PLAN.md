# Plan 02-02: Tier 1 Classification & Queue Consumer

<objective>
Implement the pgmq consumer loop scaffold with finally-block archive pattern, the Tier 1 Haiku binary classifier that reads from tier1_queue, and AI call logging to the ai_calls table. Write the tier1-classify prompt file. Set up the worker entry point with startup assertions, prompt loading, health endpoint, graceful shutdown, and the Reddit ingestion loop + Tier 1 consumer running concurrently.
</objective>

<context>
Key decisions from CONTEXT.md and RESEARCH.md that affect implementation:

- **D-10:** AI model strings pinned in config module. Haiku for Tier 1 (`TIER1_MODEL` from config.ts).
- **D-11:** Prompts live in `prompts/tier1-classify.md`, read from disk at worker startup, cached in memory.
- **D-12:** `prompt_version` computed as git short hash at startup (or `RAILWAY_GIT_COMMIT_SHA` env var fallback).
- **D-14:** Single process with concurrent async loops. This plan wires up the Reddit polling loop + Tier 1 consumer.
- **D-15:** Graceful shutdown via SIGTERM/SIGINT — set stop flag, allow in-flight to complete.
- **D-16:** Plain `http.createServer` health endpoint returning 200 OK.
- **D-17:** pgmq consumers call `archive()` in `finally` blocks. Messages exceeding 3 retries go to DLQ.
- **D-18:** Worker startup asserts all required extensions via `check_required_extensions()` RPC.
- **D-19:** Every AI call logged to `ai_calls` with model, prompt_version, tokens, cost, latency, post_id, tier.
- **D-20:** Cost computed client-side from token counts using `MODEL_PRICING` from config.

Pitfalls:
- pgmq messages need explicit `archive()` after processing or they re-deliver after visibility timeout.
- pgmq RPC functions via Supabase: `pgmq_send`, `pgmq_read`, `pgmq_archive` (underscore, not dot).
- `pgmq.read` is NOT a blocking long-poll — returns immediately with 0 messages if queue is empty.
- Tier 1 Haiku response is plain text JSON — must parse with try/catch and route parse failures to DLQ.
- Idempotency guard: check `pipeline_status` at start of tier handler — if already processed, skip and archive.
- `git rev-parse --short HEAD` may fail on Railway — fall back to `RAILWAY_GIT_COMMIT_SHA` env var.
- Prompt file path must resolve correctly in both dev (`tsx`) and prod (compiled `dist/`) — use `process.cwd()` or env var.
</context>

## Task 3: Create generic pgmq consumer loop

<read_first>
- .planning/phases/02-worker-pipeline-ingestion-classification-dedup-logging/02-RESEARCH.md (Section 2: pgmq consumer pattern, finally-block archive, DLQ routing)
- .planning/phases/02-worker-pipeline-ingestion-classification-dedup-logging/02-PATTERNS.md (consumer.ts pattern)
- apps/worker/src/config.ts (CONSUMER_BATCH_SIZE, DLQ_RETRY_THRESHOLD, visibility timeouts)
</read_first>

<action>
Create `apps/worker/src/queue/consumer.ts`:

1. **`sleep(ms: number): Promise<void>`** utility — simple setTimeout promise wrapper.

2. **`runConsumerLoop` generic consumer function:**
   ```typescript
   export async function runConsumerLoop(options: {
     db: ReturnType<typeof createClient>;
     queueName: string;
     dlqName: string;
     visibilityTimeout: number;
     batchSize: number;
     shutdown: { stop: boolean };
     processMessage: (db: ReturnType<typeof createClient>, postId: string) => Promise<void>;
   }): Promise<void>
   ```

   Implementation:
   - While `!shutdown.stop`:
     - Call `db.rpc('pgmq_read', { queue_name, vt, qty })` to read batch
     - On RPC error: log, sleep 5s, continue
     - On empty batch: sleep 2s, continue
     - For each message in batch:
       - Use the `shouldArchive` flag pattern:
         ```
         let shouldArchive = false;
         try {
           await processMessage(db, msg.message.post_id);
           shouldArchive = true;
         } catch (err) {
           if (msg.read_ct >= DLQ_RETRY_THRESHOLD) {
             await sendToDlq(db, dlqName, msg, String(err));
             shouldArchive = true;
           }
           // else: don't archive, let visibility timeout re-deliver
         } finally {
           if (shouldArchive) {
             await db.rpc('pgmq_archive', { queue_name, msg_id: msg.msg_id });
           }
         }
         ```

3. **`sendToDlq` helper:**
   - Sends the failed message payload plus error reason to the DLQ queue via `pgmq_send`
   - Logs the DLQ routing event

Key constraints:
- `shouldArchive` flag in `finally` block — never skip archive on final attempt
- Transient errors (read_ct < threshold) do NOT archive — let visibility timeout re-deliver
- Consumer checks `shutdown.stop` at top of while loop and between messages
- Sleep 2s on empty queue, 5s on RPC errors
</action>

<acceptance_criteria>
- apps/worker/src/queue/consumer.ts exists
- Exports `runConsumerLoop` function with typed options parameter
- Exports `sleep` utility function
- Uses `shouldArchive` flag pattern in finally block (never skips archive on final attempt)
- Messages exceeding retry threshold are sent to DLQ before archiving

1. **`processTier1(options)` function:**
   ```typescript
   export async function processTier1(options: {
     db: ReturnType<typeof createClient>;
     anthropic: Anthropic;
     postId: string;
     prompt: string;
     promptVersion: string;
   }): Promise<void>
   ```

   Implementation:
   a. **Idempotency guard:** Query post's `pipeline_status` — if already beyond `tier0_passed` (e.g., `tier1_rejected`, `tier2_done`), skip and return.
   b. **Fetch post content:** Read `title` and `body` from posts table.
   c. **Call Anthropic Haiku:**
      - `model: TIER1_MODEL`
      - `max_tokens: 256`
      - `messages: [{ role: 'user', content: prompt + '\n\n' + postContent }]`
      - Measure `latency_ms` as Date.now() delta
   d. **Parse response:** Extract text block, JSON.parse the content, validate with `Tier1ResultSchema.safeParse()`
   e. **Log to ai_calls** (always — both success and failure):
      - Insert row with: post_id, tier=1, model, prompt_version, input_tokens, output_tokens, cost_usd (via computeCost), latency_ms, request/response payloads, error if any
   f. **Update post record:**
      - Store `tier1_result` JSONB with `{ decision, confidence, reason, prompt_version }`
      - If `decision === 'pass'`: update `pipeline_status='tier1_passed'`, enqueue to tier2_queue
      - If `decision === 'reject'`: update `pipeline_status='tier1_rejected'`
   g. **Error handling:**
      - JSON parse failure: log to ai_calls with error, throw (routes to DLQ via consumer)
      - Zod validation failure: log to ai_calls with error, throw
      - Anthropic API error: log to ai_calls with error, throw (retryable via consumer)

Key constraints:
- Log to `ai_calls` on EVERY call (success or failure) per LOG-01
- Compute cost client-side using `computeCost` from config
- Never instantiate Anthropic client inside — accept as parameter
- Idempotency guard prevents double-processing on message re-delivery
</action>

<threat_model>
| Threat | Impact | Mitigation |
|---|---|---|
| Haiku returns markdown code fences instead of raw JSON | JSON.parse fails, message retried indefinitely | Prompt explicitly says "raw JSON only, no code fences"; parse failures throw and route to DLQ after 3 retries |
| pgmq message never archived (crash between process and archive) | Message re-delivers indefinitely, wasting Haiku calls | shouldArchive flag pattern in finally block; idempotency guard checks pipeline_status before re-processing |
| DLQ queues don't exist at consumer start | pgmq_send to DLQ fails, original message stays unarchived | Create tier1_dlq and tier2_dlq at worker startup via pgmq_create RPC (idempotent) |
| git rev-parse fails on Railway (no .git directory) | promptVersion is empty or throws | Fall back to RAILWAY_GIT_COMMIT_SHA env var; if both fail, use 'unknown' with a warning log |
| Anthropic API rate limit (429) or timeout | Consumer loop stalls | Throw on API error (retryable via consumer re-delivery); DLQ after 3 attempts prevents infinite retry |
| Concurrent duplicate messages processed (visibility timeout race) | Double AI calls, double cost | Idempotency guard on pipeline_status — second delivery skips processing and archives immediately |
| Prompt file missing at startup | Worker crashes on readFileSync | Fail fast at startup with clear path error; verification step checks file existence |
</threat_model>
