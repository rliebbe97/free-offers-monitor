# Requirements Archive: Free Offers Monitor v1.0

**Archived:** 2026-04-21
**Original defined:** 2026-04-20
**Core Value:** Reliably surface genuinely free physical goods from noisy forum data — false negatives cost missed offers, false positives erode trust.

## v1 Requirements

### Database & Infrastructure

- [x] **DB-01**: Supabase Postgres with pgvector, pgmq, and pg_cron extensions enabled and verified at startup
- [x] **DB-02**: Schema deployed with all 7 tables (sources, posts, offers, post_offers, verification_log, human_review_queue, ai_calls) and prescribed indexes
- [x] **DB-03**: Shared `@repo/db` package exports typed Supabase client, generated types, and schema SQL
- [x] **DB-04**: pgmq queues created (tier1_queue, tier2_queue) with appropriate visibility timeouts

### Ingestion

- [x] **ING-01**: Reddit adapter implements `SourceAdapter` interface with `fetchNewPosts(since: Date): Promise<RawPost[]>`
- [x] **ING-02**: snoowrap polls configured subreddits via OAuth with rate limit logging when backoff triggers
- [x] **ING-03**: Ingestion extracts top-level comments and one reply deep, skipping AutoModerator and bot accounts
- [x] **ING-04**: Deleted/removed posts and comments are detected and excluded before enqueuing
- [x] **ING-05**: Ingestion writes to `posts` table with `UNIQUE(source_id, external_id)` enforced

### Classification

- [x] **CLS-01**: Tier 0 keyword filter runs inline during ingestion with hand-maintained keyword list; rejects stored with `tier0_passed=false`
- [x] **CLS-02**: Tier 1 pgmq consumer classifies posts using Haiku binary classifier; result stored as JSONB `{decision, confidence, reason, prompt_version}`
- [x] **CLS-03**: Tier 2 pgmq consumer extracts structured offer entity using Sonnet with forced tool use (`tool_choice: { type: 'tool' }`)
- [x] **CLS-04**: Tier 2 runs exclusion checks rejecting coupons, services, non-zero shipping, trials, and sweepstakes
- [x] **CLS-05**: Tier 2 confidence < 0.7 routes to `human_review_queue` instead of auto-publishing
- [x] **CLS-06**: Tier 2 tool output validated with Zod schema before any database insert

### Deduplication

- [x] **DDP-01**: URL normalization strips UTM params and follows one redirect level before hashing with sha256
- [x] **DDP-02**: Exact URL hash match checks `offers.destination_url_hash` before creating new offer
- [x] **DDP-03**: When URL hash misses, Voyage embedding cosine similarity >= 0.85 via pgvector links to existing offer
- [x] **DDP-04**: pgvector queries run with `ivfflat.probes = 10` set in session

### Validation

- [x] **VAL-01**: pg_cron triggers daily validation checking each active offer weekly
- [x] **VAL-02**: URL liveness check via HEAD with GET fallback; 403/429 responses treated as `check_failed`, not dead
- [x] **VAL-03**: Cheerio-based dead signal detection scans page text for expiry indicators
- [x] **VAL-04**: Two consecutive failed checks 24 hours apart required before auto-expiring an offer
- [x] **VAL-05**: All validation results written to `verification_log`

### AI Logging

- [x] **LOG-01**: Every Tier 1 and Tier 2 AI call logs to `ai_calls` table with model, prompt_version (git hash), input/output tokens, cost, latency_ms, and post_id
- [x] **LOG-02**: Prompts live in `prompts/` as markdown files, versioned with git hash computed at worker startup

### Dashboard

- [x] **DSH-01**: Supabase Auth with email allowlist gates all dashboard access
- [x] **DSH-02**: Offer list page with pagination, filtering by status, and sorting
- [x] **DSH-03**: Human review queue page showing pending offers with approve/reject actions that update offer status
- [x] **DSH-04**: AI call log viewer showing cost, latency, and prompt version per call

### Worker Infrastructure

- [x] **WRK-01**: pgmq consumers call `archive()` in `finally` blocks on all code paths
- [x] **WRK-02**: Worker startup asserts all required Postgres extensions are enabled
- [x] **WRK-03**: Structured error handling with dead letter queue for messages exceeding retry threshold
