# Milestone v1.0: MVP

**Status:** SHIPPED 2026-04-21
**Phases:** 1-4
**Total Plans:** 7

## Overview

Full end-to-end pipeline for scanning Reddit for genuinely free physical goods offerings, classifying them through a three-tier AI system (keyword filter, Haiku classifier, Sonnet extractor), deduplicating via URL hashing and Voyage embeddings, validating offer liveness, and surfacing results through an auth-gated Next.js dashboard.

**Goal**: Implement the full end-to-end worker pipeline from Reddit ingestion through Tier 0 keyword filtering, Tier 1 Haiku classification, Tier 2 Sonnet extraction with deduplication, and AI call logging.
**Depends on**: Phase 1
**Plans**: 3 plans

Plans:

- [x] 02-01: Ingestion & Tier 0 — SourceAdapter interface, Reddit adapter (snoowrap OAuth), comment extraction, bot/deleted guards, posts table write, Tier 0 inline keyword filter, enqueue to tier1_queue
- [x] 02-02: Tier 1 — pgmq consumer scaffold (read + archive pattern), Haiku binary classifier, posts.tier1_result JSONB write, ai_calls logging, enqueue passes to tier2_queue
- [x] 02-03: Tier 2, Dedup & Logging — Sonnet structured extractor with forced tool use, Zod validation, exclusion checks, URL normalization + hash dedup, Voyage embedding cosine dedup, offers/post_offers writes, low-confidence routing, DLQ and error handling, prompt versioning via git hash

**Details:**
- snoowrap OAuth with rate limit logging
- Top-level comments + one reply deep, skip AutoModerator/bots
- 25 hand-maintained Tier 0 keywords
- Haiku binary classifier with JSONB result storage
- Sonnet forced tool_choice extraction with Zod validation
- Exclusion checks: coupons, services, shipping, trials, sweepstakes
- Confidence < 0.7 routes to human_review_queue
- URL normalization with one-level redirect follow + SHA-256 hash
- Voyage AI 1024-dim embeddings with pgvector cosine >= 0.85
- All AI calls logged to ai_calls table with tokens, cost, latency, prompt_version
- DLQ routing after 3 retries

## Milestone Summary

**Key Decisions:**

- Decision: Three-tier classification (keyword -> Haiku -> Sonnet) (Rationale: Balances cost and accuracy)
- Decision: pgmq over external queue (Rationale: Everything in Postgres, no extra infra)
- Decision: Voyage AI for embeddings (Rationale: Better quality for semantic dedup)
- Decision: snoowrap for Reddit API (Rationale: Most mature Node.js Reddit wrapper)
- Decision: Supabase managed platform (Rationale: Auth, Vault, pgvector, pgmq, pg_cron all-in-one)
- Decision: Next.js 16 proxy over middleware (Rationale: Newer API, same session gating pattern)
- Decision: Export runValidationCycle separately from loop (Rationale: Direct unit testing without while-loop issues)

**Issues Resolved:**

- snoowrap types incomplete — used @ts-ignore where needed per CLAUDE.md
- Supabase Relationships: [] required for postgrest-js@2.104 GenericTable
- shadcn CLI npm conflict with workspace:* protocol — temp removal workaround
- vitest.config.ts needed for config.ts env var validation at module load
- Implicit any on Supabase partial select results — explicit type casts

**Issues Deferred:**

- Discourse/forum adapters — deferred to v2
