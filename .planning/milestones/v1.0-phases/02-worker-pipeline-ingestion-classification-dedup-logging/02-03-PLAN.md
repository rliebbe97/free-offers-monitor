# Plan 02-03: Tier 2 Extraction, Dedup & Full Pipeline

<objective>
Implement Tier 2 Sonnet extraction with forced tool use, Zod validation, exclusion checks, and low-confidence routing. Implement the full deduplication pipeline: URL normalization with one-level redirect follow, SHA-256 hash matching, and Voyage AI embedding cosine similarity via pgvector. Add the find_similar_offer SQL function to the schema. Wire up the Tier 2 consumer into the worker entry point to complete the end-to-end pipeline.
</objective>

<context>
Key decisions from CONTEXT.md and RESEARCH.md that affect implementation:

- **D-13:** Tier 2 uses `tool_choice: { type: 'tool', name: 'extract_offer' }` to force tool use. Tool output validated with Zod before any DB insert.
- **D-19:** Every AI call logged to `ai_calls` — same pattern as Tier 1.
- **D-20:** Cost computed client-side.

From RESEARCH.md:
- Tier 2 response must have `stop_reason === 'tool_use'` — anything else is an error.
- Tool block extracted via `response.content.find(b => b.type === 'tool_use' && b.name === 'extract_offer')`.
- `toolBlock.input` is `unknown` — validate with `OfferExtractionSchema.safeParse()`.
- Confidence < 0.7 routes to `human_review_queue`, NOT auto-published.
- Exclusion checks: `is_excluded === true` OR `shipping_cost > 0` → reject.
- URL dedup order: hash check first (O(1)), then embedding fallback (expensive).
- `normalize-url` strips UTM params but does NOT follow redirects — custom one-level redirect follow first.
- `AbortSignal.timeout(5000)` for redirect HEAD request.
- Voyage `voyage-2` model produces 1024-dim embeddings — assert length before DB insert.
- pgvector cosine similarity threshold: 0.85.
- `ivfflat.probes = 10` set inside the `find_similar_offer` SQL function.
- On hash match or embedding match: link post to existing offer via `post_offers`.
- On no match: create new offer row + link.

Pitfalls:
- `normalize-url` v9 is ESM-only — import without `.js` suffix (npm package).
- Amazon URLs have special normalization (extract ASIN, reconstruct canonical URL).
- JS redirect shorteners (linktr.ee) return 200 with JS redirect — skip embedding dedup for these, use only hash.
- Voyage SDK may have ESM issues — have native fetch fallback ready.
- `destination_url_hash` index is not UNIQUE — handle potential races with ON CONFLICT DO NOTHING or check-then-insert.
- Schema SQL needs the `find_similar_offer` PL/pgSQL function added for pgvector RPC queries.
</context>

## Task 3: Create URL normalization and hash dedup module

<read_first>

<action>
Create `apps/worker/src/dedup/url-hash.ts`:

1. **`followOneRedirect(url: string): Promise<string>`:**
   - Sends HEAD request with `redirect: 'manual'` and `AbortSignal.timeout(5000)`
   - If response is 3xx and has `location` header, return the location
   - On any error (network, timeout): return original URL (silent fallback)
   - Do NOT recurse into the resolved URL

2. **`normalizeAmazonUrl(url: string): string`** (internal helper):
   - Extracts ASIN from Amazon URLs (`/dp/ASIN`, `/gp/product/ASIN`)
   - Returns canonical `https://www.amazon.com/dp/{ASIN}` if found
   - Returns original URL if not an Amazon URL

3. **`normalizeAndHash(rawUrl: string): Promise<{ normalizedUrl: string; hash: string }>`:**
   - Step 1: `followOneRedirect(rawUrl)`
   - Step 2: `normalizeAmazonUrl(resolved)` (if Amazon URL)
   - Step 3: `normalizeUrl(resolved, { ... })` with options:
     - `stripWWW: false`
     - `removeQueryParameters: [/^utm_/i, 'ref', 'source', 'fbclid', 'gclid']`
     - `sortQueryParameters: true`
     - `stripHash: true`
     - `normalizeProtocol: true`
   - Step 4: SHA-256 hash of the normalized URL
   - Return `{ normalizedUrl, hash }`

4. **`findExistingOfferByHash(db, urlHash: string): Promise<string | null>`:**
   - Queries `offers` table for `destination_url_hash === urlHash`
   - Returns offer `id` if found, `null` if not
   - Uses `.limit(1).maybeSingle()` (not `.single()` which throws on no match)

All functions are named exports.
</action>

   - Model: `'voyage-2'` (explicitly pinned)

   ```typescript
   export async function embedText(text: string): Promise<number[]> {
     const response = await fetch('https://api.voyageai.com/v1/embeddings', {
       method: 'POST',
       headers: {
         'Authorization': `Bearer ${process.env.VOYAGE_API_KEY}`,
         'Content-Type': 'application/json',
       },
       body: JSON.stringify({ input: [text], model: 'voyage-2' }),
     });
     if (!response.ok) throw new Error(`Voyage API error: ${response.status}`);
     const data = await response.json() as { data: Array<{ embedding: number[] }> };
     const embedding = data.data[0]?.embedding;
     if (!embedding || embedding.length !== 1024) {
       throw new Error(`Expected 1024-dim embedding, got: ${embedding?.length ?? 'null'}`);
     }
     return embedding;
   }
   ```

   - Format embedding as vector string: `[${embedding.join(',')}]`
   - Returns offer `id` if match found above threshold, `null` otherwise

Named exports only. Uses native fetch (not voyageai SDK) to avoid ESM compatibility issues.

Add the `find_similar_offer` function to `packages/db/src/schema.sql`:

Insert a new section between the EXTENSION VERIFICATION section and the QUEUES section:

```sql
-- ============================================================
-- DEDUP FUNCTIONS
-- ============================================================
-- Used by the worker dedup pipeline via db.rpc('find_similar_offer', ...).
-- Sets ivfflat.probes within the function for consistent ANN recall.

CREATE OR REPLACE FUNCTION find_similar_offer(
  query_embedding vector(1024),
  similarity_threshold float,
  match_count int
)
RETURNS TABLE(id uuid, similarity float)
LANGUAGE plpgsql
AS $$
BEGIN
  -- Set probes for this transaction (DDP-04)
  PERFORM set_config('ivfflat.probes', '10', true);

  RETURN QUERY
  SELECT
    offers.id,
    (1 - (offers.embedding <=> query_embedding))::float AS similarity
  FROM offers
  WHERE offers.status = 'active'
    AND offers.embedding IS NOT NULL
    AND (1 - (offers.embedding <=> query_embedding)) >= similarity_threshold
  ORDER BY offers.embedding <=> query_embedding
  LIMIT match_count;
END;
$$;
```

Key details:
- `CREATE OR REPLACE FUNCTION` — idempotent, safe to re-run
- `vector(1024)` matches the offers.embedding column type
- `ivfflat.probes = 10` set inside the function (DDP-04)
- Only searches `status = 'active'` offers with non-null embeddings
- Uses cosine distance operator `<=>`
- Returns table with `id` and `similarity` score
</action>

<threat_model>
| Threat | Impact | Mitigation |
|---|---|---|
| Sonnet does not call the tool despite tool_choice forcing | No extraction, message retried indefinitely | Assert `stop_reason === 'tool_use'`; throw on mismatch which routes to DLQ after 3 retries |
