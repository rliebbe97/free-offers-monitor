### `apps/worker/src/queue/consumer.ts` (create)
**Role:** Generic pgmq consumer loop — finally-block archive pattern, DLQ routing, retry counting.
**Closest analog:** No direct analog exists yet; pattern derived from RESEARCH.md section 2.
**Pattern to follow (from RESEARCH.md — the established pattern):**
```typescript
for (const msg of messages) {
  let shouldArchive = false;
  try {
    await processTier1(db, msg.message.post_id);
    shouldArchive = true;
  } catch (err) {
    if (msg.read_ct >= 3) {
      await sendToDlq('tier1_dlq', msg, String(err));
      shouldArchive = true;  // archive after DLQ
    }
    // else: don't archive, let visibility timeout re-deliver
  } finally {
    if (shouldArchive) {
      await db.rpc('pgmq_archive', { queue_name: 'tier1_queue', msg_id: msg.msg_id });
    }
  }
}
```
**Key conventions:**
- `shouldArchive` flag pattern in finally block — never skip `pgmq_archive` on the final attempt
- `msg.read_ct >= 3` for DLQ routing — check this constant matches RESEARCH.md decision
- Sleep on empty queue (2s) and on RPC errors (5s) — use a `sleep` helper from a shared util
- Accept `shutdown: { stop: boolean }` object — check flag at top of each while-loop iteration
- Named exports only: `export async function runTier1Consumer(...)`, `export async function runTier2Consumer(...)`

## Cross-Cutting Patterns

### Module resolution
- All internal imports use `.js` extension (NodeNext resolution): `import { foo } from './foo.js'`
- External npm packages do not use extensions: `import Anthropic from '@anthropic-ai/sdk'`
- `@repo/db` is the only internal package import used in the worker: `import { createClient } from '@repo/db'`

### Named exports
- All files use named exports only — no `export default`
- Re-export barrel files (`index.ts`) list each export explicitly with `export { ... }` or `export type { ... }`

### No `any`
- Use `unknown` for untrusted external data (API responses, JSON parse results, tool block input)
- Use type narrowing (`typeof`, `instanceof`, Zod `safeParse`) before accessing fields on `unknown`
- `@ts-ignore` is only permitted at the snoowrap adapter boundary, always with a justifying comment

### Environment variables
- Pattern from `packages/db/src/client.ts`: read env var, check for undefined, throw with a message naming the variable
- Never use `!` non-null assertion on env vars — always check explicitly
- Never fall back to empty string — fail fast with a clear error

### DB client injection
- The `createClient()` factory from `@repo/db` is called once in `index.ts`
- The resulting `db` instance is passed as a parameter to every function that needs DB access
- Type of `db` is always `ReturnType<typeof createClient>` — never re-typed or cast

### Error handling
- Supabase query results: always destructure `{ data, error }` and check `error` before using `data`
- Consumer loops: transient errors do not archive the pgmq message (let visibility timeout re-deliver)
- Fatal startup errors: throw from `main()`, caught by `.catch(() => process.exit(1))`
- AI call failures: logged to `ai_calls` with the `error` field populated; do not swallow

### TypeScript strict mode flags (from `base.json`)
- `strict: true` — enables all strict checks
- `noUncheckedIndexedAccess: true` — array/object indexing returns `T | undefined`
- `isolatedModules: true` — no `const enum`, no namespace merging

### Build tooling
- All packages use `tsup src/index.ts --format esm --dts` as the build script
- Output goes to `dist/` — excluded from TypeScript compilation via `tsconfig.json`
- Turborepo task graph: `build` depends on `^build` (packages build before apps)

### Testing
- Vitest is the test runner across all packages
- Test command: `pnpm test` (root) or `pnpm test --filter worker` (worker only)
- Tests live alongside source (e.g., `src/tiers/tier0.test.ts`) or in a `__tests__` subdirectory — follow whichever the first test file establishes
